# 픽스 IT 연구소 자동화 지침

## 블로그 정체성
- 이름: 픽스 IT 연구소 (fixitlab-kr)
- 언어: 한국어
- 정체성: IT·AI가 어렵게 느껴지는 일반인을 위한 쉬운 디지털 생활 가이드
- 페르소나: 어려운 기술을 옆집 이웃에게 쉽게 설명해주는 친절한 가이드
- 헤더: "어려운 IT·AI, 쉽게 풀어드립니다. 일상에서 바로 써먹는 디지털 생활 가이드"

## 대시보드 구조
- 실시간 급상승 관심 주제 10개 + 수익형 스테디 가이드 10개 = 총 20개
- 카테고리 구분 없이 IT·AI 전체 영역에서 선정
- 전문가 대상 주제 금지
- 에러 코드·전문 기술 용어 단독 주제 금지

## 트렌드 수집 출처
- [실시간]: 네이버 자동완성·연관검색어 (IT·AI·스마트폰 급상승),
  구글 트렌드 한국 IT·AI 카테고리,
  네이버 지식IN ("~란 무엇인가요?" 다수 추천 패턴),
  유튜브 IT·AI 쉬운 설명·생활팁 급상승 영상
  → 최근 24~48시간 기준
- [스테디]: 구글 자동완성·People Also Ask (한국어),
  Semrush/Ahrefs IT 입문 롱테일 키워드,
  네이버 블로그 상위 노출 IT·AI 인기글 패턴
  → 최근 6개월 기준

## 주제 유형
- 스마트폰·PC 생활 활용 팁
- AI 도구 쉬운 입문 가이드
- IT 용어 쉬운 설명
- 앱·서비스 비교·추천
- 디지털 보안·개인정보 보호 팁

## 제목 생성 규칙
- 글자 수: 공백 포함 45~60자
- 검색 의도 반영: 일반인 궁금증 키워드를 제목 앞부분에 배치
- 클릭 유도 패턴 (주제에 맞게 선택):
  · 궁금증 해소형: "~란 무엇인가요? 5분이면 이해돼요"
  · 생활 연결형: "~을 알면 이렇게 달라집니다"
  · 쉬운 설명형: "전문가 없이도 이해되는 ~"
  · 비교·추천형: "~ vs ~, 뭐가 더 나을까요?"
  · 수치·단계형: "3가지 방법", "5분이면 OK", "2026년 최신"
- 전문 기술 용어 단독 제목 금지
- 생활 연결 표현 포함 필수

## 퍼머링크 생성 규칙
- 소문자 영문 + 하이픈 연결
- 최대 60자
- 본문 핵심 키워드 반영
- 예: "galaxy-ai-translation-feature-guide-2026"

## 글 구조 (순서 엄수)
1. 핵심 요약 박스 (✅ 3줄)
2. 공감형 도입부 (일반인 공감 상황, 기술 나열 금지)
3. 이미지1
4. 쉬운 개념 설명 (비유·일상 예시 우선)
5. 핵심 내용 (단계별 가이드·활용법, <ol><li>)
6. 비교표 (HTML table)
7. 이미지2
8. 실생활 활용 및 꿀팁
9. FAQ 3개 이상
10. 이미지3
11. 결론 (관련 주제 탐색 유도)

## 톤 & 말투
- 친근하고 쉬운 어조 — "옆집 이웃에게 설명하듯"
- 전문 용어 첫 등장 시 반드시 비유로 먼저 설명
  예: "이건 마치 ~과 같은 원리예요"
- 본문 중간 환기 문장 1회 필수
